2. 데이터 전처리 중요

Numpy / Pandas

도구핵심 기능주요 함수
Numpy배열·벡터·행렬 연산np.array, A @ B, A.T
Pandas표 형태 데이터 분석df.shape, df.info(), pd.read_csv()

텍스트 전처리 순서

Cleaning→Regex→Tokenization→Normalization→Stopwords 제거
Cleaning: HTML, URL, 이모지, 특수문자 제거
Regex: 패턴 기반 검색·치환. \d+ 숫자, [가-힣]+ 한글, [a-zA-Z]+ 영문
Tokenization: 문장/단어/형태소 단위 분리
Normalization: 대소문자 통일, 숫자 치환, 반복문자 축소
Stopwords: 의미 없는 조사·접속사 제거

텍스트 수치화

  • Bag of Words: 단어 출현 횟수 벡터화. 빠르지만 순서 정보 손실
  • TF-IDF: 단어 빈도 × 역문서 빈도. 흔한 단어의 중요도는 낮춤
  • 문서 유사도: 코사인 유사도는 방향, 유클리드 거리는 직선 거리

3. LLM 핵심 최중요

임베딩

단어·문장 같은 추상적 데이터를 컴퓨터가 계산할 수 있는 Dense Vector로 바꾸는 과정입니다.

Transformer

Transformer = Attention 기반 Seq2Seq 모델
Encoder는 입력 문장의 의미를 벡터로 계산하고, Decoder는 그 결과를 바탕으로 목표 문장을 순차 생성합니다.
개념한 줄 정의시험 포인트
Positional Encoding토큰 위치 정보를 추가Transformer는 순서를 모르므로 필요
Self-Attention각 토큰이 다른 토큰과의 중요도를 계산문맥 반영의 핵심
Multi-Head Attention여러 Attention을 병렬 수행다양한 관계를 학습
Masked Self-Attention미래 토큰을 보지 못하게 차단생성 모델에서 필수
FFNNAttention 뒤의 비선형 변환표현력 강화

Encoder vs Decoder

구분역할대표 모델
Encoder문장 이해, 양방향 문맥 참조BERT
Decoder다음 토큰 예측, 이전 토큰만 참조GPT

GPT

GPT의 핵심 원리 = Next Token Prediction

주어진 문맥에서 다음 토큰을 예측하고, 생성된 토큰을 다시 입력에 추가하여 이어서 생성합니다. 이를 자기회귀적(Autoregressive) 생성이라고 합니다.

LLM 최적화·발전

  • MoE: 여러 전문가 MLP 중 일부만 활성화
  • Quantization: 가중치를 INT8/INT4 등으로 압축해 VRAM 절약
  • sLLM: 작은 모델을 증류·가지치기·파인튜닝으로 특화
  • Tool Calling / Agent: 외부 API·검색·코드 실행 도구를 호출해 답변

5. RAG 최중요

RAG가 필요한 이유
LLM은 최신 데이터, 내부 데이터, 도메인 특화 데이터에 약할 수 있습니다. 관련 문서를 검색해서 Context로 넣으면 할루시네이션을 줄일 수 있습니다.

RAG 5-step Process

Indexing→Query Processing→Searching→Augmenting→Generating
단계내용
Indexing문서를 Chunking하고 Embedding 후 DB에 저장
Query Processing질문을 검색하기 좋게 전처리·재구성
SearchingSemantic, Lexical, Hybrid Search 수행
Augmenting검색 결과를 프롬프트 Context에 추가
GeneratingLLM이 근거 기반 답변 생성

검색 방식

  • Semantic Search: 임베딩 기반 의미 검색. 상담·고객문의에 강함
  • Lexical Search: BM25/TF-IDF 기반 키워드 검색. 법령·조항·모델명에 강함
  • Hybrid Search: 두 검색을 결합해 정확도 보완

고급 RAG 기법

  • Chunk Overlap: 청크 간 일부 문맥을 겹쳐 의미 보존
  • Small2Big / Sliding Window: 작은 청크로 검색하고 주변 큰 문맥 전달
  • Contextual Retrieval: 청크 앞에 전체 문맥 기반 헤더 추가
  • HyDE: 질문에 대한 가상 답변/문서를 만든 뒤 검색
  • Query Reformulation: 하나의 질문을 여러 질문으로 바꿔 다중 검색
  • Reranker: 검색된 청크를 다시 점수화해 재정렬
  • GraphRAG: 문서를 그래프 형태로 저장·검색
  • Multimodal RAG: 이미지·표·텍스트를 함께 검색

RAG 평가

Retrieval과 Generation을 따로 평가해야 합니다. 주요 지표는 Faithfulness, Factual Correctness, Context Recall, BLEU, Semantic Similarity입니다.

6. LLM Fine-Tuning 중요

Fine-Tuning은 이미 학습된 모델에 추가 데이터를 학습시켜 특정 목적의 성능을 높이는 방법입니다.

기법핵심 개념특징
SFT정답 데이터를 그대로 학습가장 기본적인 방식
Continuous Pretraining도메인/언어 코퍼스 추가 학습지식 주입, 최신 정보 반영
Instruction Tuning질의응답·지시 데이터 학습명령 수행 능력 개선
RAFTRAG 형식 데이터로 학습문서 근거 기반 답변 강화
RLHF인간 피드백 기반 보상 학습선호 반영, Alignment
DPO선호 쌍으로 직접 최적화PPO보다 단순·안정적
LoRA저차원 행렬 어댑터 추가대표 PEFT, 메모리 절약
DoRAWeight를 크기/방향으로 분해LoRA 개선
주의할 점
Fine-Tuning은 Catastrophic Forgetting과 Overfitting을 유발할 수 있습니다. 새 지식을 배우는 대신 기존 능력이 줄어들 수 있으므로 데이터 구성과 평가가 중요합니다.

LoRA 암기

LoRA = 적은 파라미터만 학습해서 빠르고 저렴하게 튜닝

  • 기존 모델 가중치를 직접 바꾸지 않음
  • A/B 테스트와 버전 관리가 쉬움
  • Full Fine-Tuning보다 학습 효과가 약할 수 있음
